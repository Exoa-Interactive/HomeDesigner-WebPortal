HomeDesigner Web Portal

A full-stack web portal for managing and previewing 3D home design projects — built on Laravel 12, React 18, and Inertia.js.

Laravel 12 React 18 Inertia.js Three.js Tailwind CSS Vite 7 SQLite / MySQL

🔍Overview

The HomeDesigner Web Portal is a project library application that lets users browse, upload, and interactively preview 3D home design models. It serves two audiences:

  • End users — browse the public gallery of 3D design projects, filter by type, and preview models in an interactive 3D viewer.
  • Administrators — log in to create, edit, and delete projects, uploading GLB 3D model files, JSON metadata, and cover images.
  • External clients / apps — integrate via the REST API to programmatically list, create, or update projects using bearer-token authentication.
ℹ️
Hybrid Architecture
The portal uses Inertia.js so the web interface is a true SPA — no page reloads — while the backend is a conventional Laravel app that renders React components server-side. It also exposes a separate REST API at /api/* for external integrations.

Request Lifecycle (Web)

  1. Browser navigates to a URL.
  2. Laravel handles the request via routes/web.php.
  3. Controller fetches data from the database using Eloquent.
  4. Inertia renders the matching React page component, injecting data as props.
  5. On subsequent navigation, Inertia intercepts links and fetches only the new page's JSON (no full reload).

Request Lifecycle (API)

  1. Client sends request to /api/* with optional Authorization: Bearer <token> header.
  2. Laravel Sanctum validates the token (if protected route).
  3. API Controller processes the request and returns JSON.

🔒Authentication

Web (Session-based)

The admin interface uses Laravel's standard session authentication. A user logs in via the /login form, and Laravel sets a signed session cookie. All admin routes are protected with the auth middleware.

API (Token-based — Laravel Sanctum)

External clients authenticate with bearer tokens:

1. Obtain a token
POST /api/login
Content-Type: application/json

{
  "email": "admin@example.com",
  "password": "your-password"
}

// Response
{
  "token": "1|abc123...",
  "user": { "id": 1, "name": "Admin", ... }
}
2. Use the token in subsequent requests
GET /api/my-projects
Authorization: Bearer 1|abc123...
Accept: application/json
⚠️
Security Note
The public API project endpoints (list, create, update, delete) do not require authentication by default. Restrict these routes if the portal should be access-controlled in production.

Project Types

  • template — shared design templates available to all users.
  • userfile — projects belonging to a specific user account.

🎓3D Viewer

The ProjectPreviewModal component renders GLB files directly in the browser using Three.js. It is opened when a user clicks the "3D Preview" button on any project that has a GLB file.

Features

  • Loads GLB models via GLTFLoader
  • Orbit controls — click + drag to rotate, scroll to zoom, right-click to pan
  • Professional 4-light setup (directional lights + ambient) for realistic shading
  • Plays embedded model animations via AnimationMixer
  • Shadow toggle button
  • Reset camera button
  • Loading spinner and error state handling
ℹ️
Three.js is a production dependency (package.json > dependencies), not a dev dependency. It must be included in the production bundle — npm run build handles this automatically.

⚙️Nginx Configuration

/etc/nginx/sites-available/homedesigner
server {
    listen 80;
    listen [::]:80;
    server_name yourdomain.com www.yourdomain.com;

    # Redirect all HTTP to HTTPS
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    listen [::]:443 ssl http2;
    server_name yourdomain.com www.yourdomain.com;

    root /var/www/homedesigner/public;
    index index.php;

    # SSL (use Certbot / Let's Encrypt)
    ssl_certificate /etc/letsencrypt/live/yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/yourdomain.com/privkey.pem;

    # Increase upload limits for GLB files (50 MB)
    client_max_body_size 55M;

    # All requests go to index.php (Laravel front controller)
    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    # PHP-FPM handler
    location ~ \.php$ {
        fastcgi_pass unix:/var/run/php/php8.2-fpm.sock;
        fastcgi_index index.php;
        fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
        include fastcgi_params;
    }

    # Block access to hidden files (.env, .git, etc.)
    location ~ /\.(?!well-known).* {
        deny all;
    }

    # Cache static assets
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|glb)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
# Enable and reload
ln -s /etc/nginx/sites-available/homedesigner /etc/nginx/sites-enabled/
nginx -t && systemctl reload nginx
🔑
Free SSL with Certbot
apt install certbot python3-certbot-nginx && certbot --nginx -d yourdomain.com

⚙️Apache Configuration

/etc/apache2/sites-available/homedesigner.conf
<VirtualHost *:80>
    ServerName yourdomain.com
    ServerAlias www.yourdomain.com

    DocumentRoot /var/www/homedesigner/public

    # Increase file upload size for GLB models
    php_value upload_max_filesize 55M
    php_value post_max_size 60M

    <Directory /var/www/homedesigner/public>
        Options -Indexes +FollowSymLinks
        AllowOverride All
        Require all granted
    </Directory>

    # Block access to .env files
    <FilesMatch "^\.env">
        Require all denied
    </FilesMatch>

    ErrorLog ${APACHE_LOG_DIR}/homedesigner_error.log
    CustomLog ${APACHE_LOG_DIR}/homedesigner_access.log combined
</VirtualHost>
# Enable required modules and site
a2enmod rewrite
a2ensite homedesigner
systemctl reload apache2
ℹ️
mod_rewrite required
The public/.htaccess file included with Laravel handles URL rewriting. Make sure mod_rewrite is enabled and AllowOverride All is set.

🔧Troubleshooting

500 Internal Server Error

# Check the Laravel log
tail -f storage/logs/laravel.log

# Make sure storage is writable
chmod -R 775 storage bootstrap/cache
chown -R www-data:www-data storage bootstrap/cache

Assets not loading (404 on /build/*)

# Rebuild Vite assets
npm run build

# Make sure APP_URL in .env matches the domain exactly
APP_URL=https://yourdomain.com

Uploaded files not accessible (/storage/* returns 404)

# Recreate the storage symlink
php artisan storage:link

File upload fails with 413 error

Your web server or PHP is rejecting large files. For Nginx:

client_max_body_size 60M;  # in nginx.conf or server block

For PHP (php.ini):

upload_max_filesize = 55M
post_max_size = 60M

Route cache stale after code changes

php artisan route:clear
php artisan config:clear
php artisan cache:clear

Blank page (Inertia / React not rendering)

  • Run npm run build and check for build errors.
  • Confirm public/build/manifest.json exists.
  • Check the browser console for JavaScript errors.
  • Ensure APP_DEBUG=true temporarily to see server errors.
